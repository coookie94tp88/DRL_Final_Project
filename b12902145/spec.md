### 1. 遊戲基本設定 (Overview)

* **環境名稱：** OracleGambit 


* **核心概念：** 資訊不對稱與動態賠率的博弈 。


* **場景設置：** 遊戲中有 **4** 扇門，但只有 **1** 扇門背後有真正的獎勵 。

 
* **參與人數：** **1** 位 Host (莊家/神諭者) 與 **10** 位 Players (玩家) 。


* **初始資金：** 每位 Player 初始擁有 **1000** 元的本金 (`INITIAL_BALANCE`) 。


### 2. 核心角色與目標 (Roles & Objectives)

* **The Host (莊家)：**
 
    * **資訊優勢：** 打從一開始就知道哪一扇門是正確的 。
    
    * **目標：** 最大化自己的總利潤。利潤來源於「玩家的總下注金」與「玩家的賄賂金」，並需要扣除「支付給贏家的獎金」 。
    
    * **困境：** 為了賺取手續費與賄賂，Host 需要適度透露真實資訊以維持玩家的信任；但如果太多玩家猜中，Host 就會面臨破產 。


* **The Players (玩家)：**

    * **資訊劣勢：** 不知道哪扇門是正確的，只能仰賴 Host 釋放的真假訊號。 

    * **目標：** 最大化自己的資產。必須判斷 Host 的訊號可信度，並決定下注目標與金額 。


### 3. 遊戲兩階段流程 (Two-Phase Sequence)

每一回合 (Round) 都嚴格遵守以下兩個階段：

* **Phase I: 賄賂與訊號階段 (The Bribery Stage)**

    1. **賄賂：** 每位 Player 決定要支付多少 `Bribe_Amount` 給 Host 。

    2. **發布訊號：** Host 收到所有人的賄賂後，會發布一個**公開訊號** (Public Signal，廣播給所有人，可能是假的) 。同時，Host 會根據每位 Player 給的賄賂金額，私下發布**私人訊號** (Private Signal) 給該 Player 。

* **Phase II: 下注與結算階段 (The Betting Stage)**
    1. **下注：** Player 根據收到的訊號，決定要選擇哪一扇門 (1-4)，以及要下注多少金額 。

    2. **結算：** 揭曉正確的門，系統根據「動態賠率」計算贏家獎金，並更新所有人的餘額 。


### 4. 動態賠率結算機制 (Dynamic Payout Engine)

這是這個遊戲最核心的數學引擎，用來防止所有人都無腦跟單。

* **參數定義：** 

    * $P$：該回合所有玩家的**總下注池 (Total Pool)** 。

    * $W$：押在正確門上的**總贏家下注量 (Winning Volume)** 。

    * $x$：**勝率比例 (Winning Ratio)**，公式為 $x = \frac{W}{P}$ 。

    * $\theta$：**Host 損益兩平的閾值**，設定為 **25%** ($0.25$) 。

    * **賠率乘數 $M(x)$：** 獎金不是固定的，而是根據有多少人贏來動態決定。公式為：

        $$M(x) = 1 + \frac{1 - \theta}{x}$$
    
        (這代表越少人猜中，贏家獲得的倍率就越高；越多人猜中，倍率就越低 )


* **Host 損益計算：**

    $$R_{host} = (\text{總下注池} P) - (\text{贏家總獎金}) + (\text{總賄賂金})$$

    (如果 $x < 25\%$，Host 賺錢；如果 $x = 25\%$，Host 不賺不賠；如果 $x > 25\%$，Host 將面臨嚴重虧損 )
